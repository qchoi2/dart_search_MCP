# 단계 5 캐시·동시성 성능시험 결과

측정일: 2026-07-17 (Asia/Seoul)

## 제한 라이브 측정

- OpenDART 요청: 총 16회(목록 4회, 원문 12회), TLS 호스트명·인증서 검증 유지
- 목록 2개: 순차 0.783초 → 동시성 2에서 0.398초, 49.1% 단축, 오류 0건
- 원문 6개: 순차 1.622초 → 동시성 3에서 0.595초, 63.3% 단축, 오류 0건
- DART 웹 본문검색은 측정·변경하지 않았으며 동시성 1, 요청 시작간격 1,000ms를 유지한다.

이 결과로 OpenDART 목록 상한 2와 원문 상한 3을 활성화한다. HTTP 429·일시적 timeout·5xx가 관찰되면 원문 동시성을 3→2→1로 낮춘다. OpenDART 상태 020은 감속·재시도하지 않고 즉시 중단한다.

6개 실제 정규화 원문과 순차 다운로드 1.622초를 최초 검색 기준시간으로 사용했다.

활성화 기준은 최초 검색시간 증가 5% 이하, 문서당 쓰기 p95 100ms 이하이다. B와 C가 모두 통과했고 디스크 사용량이 더 작은 C(gzip level 1)를 기본으로 선택했다. 손상 gzip/checksum 파일은 예외를 외부로 전파하지 않고 삭제 후 cache miss로 복구하는 시험도 통과했다.

## 판정 범위

이번 측정은 기능 활성화용 제한 프로브다. 여러 시간대·다양한 네트워크에서의 장기 p95나 “모든 표준 검색 90초 이내”를 보장하는 측정은 아니다. 오류 시 자동 감속과 hard deadline은 별도 fixture 회귀로 고정한다.
