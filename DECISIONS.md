# Stage 0 Decisions

실측 결과와 후속 사용자 결정을 기록한다. 미확인 사실과 개인용 사용 결정을 구분한다.

## 1. DART 본문검색 개인용 조건부 사용

- 실제 결과: robots.txt에서 /dsab007 금지는 확인되지 않았고 순차 요청은 성공했으나, 자동화 허용 빈도와 명시적 허용 문구는 확인하지 못함
- 변경: 개인용 로컬 환경에서는 기술 상태가 정상이고 검색 품질 향상이 확인되면 동시성 1·요청 시작간격 최소 1,000ms로 조건부 사용
- 이유: 사용자가 개인용 범위에서 정책 미확인 위험을 수용했으며, 명시적 금지·접근거부·반복 구조장애 시에는 OpenDART로 폴백함

## 2. 단계 0 게이트 분리

- 실제 결과: OpenDART 관련 항목과 DART 정상 파서는 통과했지만 DART 정책과 실제 구조장애는 미확인
- 변경: OpenDART 목록·정정규칙·D004·DART 파서·DART 개인용·DART 정책 게이트를 분리
- 이유: DART의 미확인 항목이 OpenDART 기반 개발 전체를 차단하지 않도록 함

## 3. 정정 사건 그룹 키

- 실제 결과: 같은 회사·기간·정규화 보고서명에도 서로 독립적인 공시 사건이 함께 반환된 실제 사례가 확인됨
- 변경: 명시적 원접수번호·정정대상 접수번호·정정사항표를 우선 사용하고 회사·보고서명·시간 군집은 보조수단으로만 사용
- 이유: 아이엠증권의 last_reprt_at=Y 응답에 비교 대상 체인 밖의 동일 보고서명 공시가 별도로 존재함

## 4. last_reprt_at 사용범위

- 실제 결과: 측정한 3개 사건에서 N은 체인 전체, Y는 해당 사건 최종 접수를 반환했지만 Y에 독립 사건이 함께 존재함
- 변경: N은 체인 수집, Y는 후보 축소에만 사용하고 사건 식별키나 최종본 증명으로 사용하지 않음
- 이유: 동일 회사·기간·정규화 보고서명이 사건의 유일성을 보장하지 않음

## 5. rm 표본 신뢰도

- 실제 결과: 정 28건·철 2건이 후속 문서와 일치했으나 검증에 회사·보고서명·근접기간 휴리스틱이 포함됨
- 변경: rm은 후보 신호로만 사용하고 명시적 접수번호·정정사항표·후속 공시로 최종 검증
- 이유: 철 표본이 적고 휴리스틱이 독립 사건을 혼합할 수 있음

## 6. 연장결정 접두어 검색 유형

- 실제 결과: [연장결정]은 B 계열 주요사항보고서(자기주식취득신탁계약체결결정)에서 실제 관측됨
- 변경: 연장결정 표본 탐색을 E002가 아니라 B 공시유형으로 라우팅
- 이유: 실제 OpenDART 목록 JSON의 report_nm과 pblntf_ty=B 조건으로 확인

## 7. D004 사건 단위와 대상회사 신뢰도

- 실제 결과: corp_name 대상회사 일치는 공시 10건에서 확인됐지만 회사·제출인 조합은 5개
- 변경: 공시 건수와 공개매수 사건 수를 분리하고 원문 생략 시 출처·미검증 상태·신뢰도를 표시
- 이유: 같은 사건의 신고서·설명서·결과보고서가 별도 공시로 반환될 수 있음

## 8. 013 적용범위

- 실제 결과: 목록 API D004 빈 기간창에서 013과 목록 부재를 확인
- 변경: 목록 API의 013을 정상 빈 배열로 처리하되 다른 엔드포인트까지 실측됐다고 일반화하지 않음
- 이유: 실측 조건과 엔드포인트의 범위를 규칙 신뢰도에 반영

## 9. DART 구조장애 판정

- 실제 결과: 정상 결과와 정상 0건은 확인했지만 실제 구조장애는 관찰하지 못함
- 변경: 첫 모호 응답은 structure_failure_candidate로 두고 상태진단 재시도에서도 반복될 때만 구조장애로 승격
- 이유: 합성 fixture는 분류기 분기 검증일 뿐 실제 장애의 증거가 아님
