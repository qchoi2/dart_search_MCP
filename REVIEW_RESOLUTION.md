## 직접 확인할 사항

1. DART 서버 세션이 실제로 만료·교체될 때 응답 Cookie와 `/search.ax` 본문이 어떻게 변하는지 별도 제한 프로브로 확인한다.
2. OpenDART 목록·원문 동시성 2/3과 감속은 라이브 요청 간격·429/5xx 표본을 포함한 성능 단계에서 확인한다.
3. 단계 2 응답 계약을 열 때 `DART_FULLTEXT_FALLBACK` 구조 경고와 완전성 등급을 확정한다.

## 남은 위험

- hard timeout은 후보 검증 반복 사이에서 집행된다. 진행 중인 단일 HTTP 요청은 공통 HTTP timeout까지 반환을 기다린다.
- `채` 포함 실제 조합문자 표본은 여전히 0건이므로 조합 신뢰도는 unconfirmed다.
- 정정 체인·동일 사건 연결은 의도적으로 수행하지 않는다. 같은 사건의 여러 정정 접수번호가 각각 결과를 차지할 수 있다.
- 순차 OpenDART 실행은 안전하지만 40문서 검증의 90초 목표를 라이브 환경에서 보장하지 않는다.
