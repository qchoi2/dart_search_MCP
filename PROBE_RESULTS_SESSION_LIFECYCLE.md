실제 서버 세션 만료는 제한된 요청 안에서 발생시키거나 관찰하지 못했다. 따라서 서버 만료 신호는 `unconfirmed`이고 자동감지는 구현하지 않았다. 런타임은 클라이언트가 확실히 아는 `reset_session`, Cookie jar 재생성, 프로세스 재시작, 검색모드 변경만으로 상태를 폐기한다.

## 관찰 결과

1. 새 Cookie 세션의 `main.do`, 모드설정 POST, 결과요청은 모두 HTTP 200이었다.
2. 같은 세션·같은 본문검색 모드에서 검색어를 바꾼 직접 `search.ax` 요청은 정상 결과행을 반환했다.
3. Cookie jar를 명시적으로 재생성한 뒤 모드설정 없이 보낸 직접 `search.ax`도 정상 결과행을 반환했고 새 Cookie 발급 헤더가 관찰됐다.
4. 새 jar에서 모드설정 POST를 수행한 뒤의 같은 검색 응답은 모드설정 전 직접 검색과 본문 SHA-256, 검색건수 15, 결과행 10개가 모두 같았다.

이 관찰은 “Cookie jar 교체가 항상 식별 가능한 실패 응답을 만든다”는 계약을 지지하지 않는다. 새 Cookie 발급 헤더 자체도 정상 신규 세션에서 나타나므로 만료 신호로 사용할 수 없다. 특정 Cookie 이름, 로그인 페이지 추정 문자열 또는 비고정 HTML을 자동 만료 신호로 하드코딩하지 않는다.

## 보존 산출물

- 최소 golden fixture는 상태코드, 구조 분류, 건수, Cookie 존재 여부와 응답 해시만 보존한다.
- Cookie 이름·값, 요청 헤더 원문, HTML 원문은 저장하지 않는다.
- 서버 만료가 실제로 재현되기 전까지 자동 만료 감지는 `not_implemented`로 유지한다.
