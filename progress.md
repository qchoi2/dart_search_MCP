## 실제 실측 결과

### 1. `last_reprt_at=N/Y`

다중 정정이 발생한 실제 사건 3건을 확인했다.

- 한진 `증권신고서(채무증권)`
- 아이엠증권 `주요사항보고서(자본으로인정되는채무증권발행결정)`
- 지에스엔텍 `증권신고서(채무증권)`

세 사건 모두 N에서는 원공시·중간 정정·최종 정정이 반환됐고, Y에서는 해당 사건의 최종 접수만 남았다. 다만 아이엠증권 사례에서 같은 회사·기간·정규화 보고서명에 속하는 별도 사건도 Y 응답에 함께 존재했다. 따라서 회사와 보고서명만으로 정정 사건을 묶으면 안 된다는 결정을 `DECISIONS.md`에 기록했다.

### 2. `rm`의 `정`·`철`

- 서로 다른 사건 30건 검증
- `정` 28건, `철` 2건
- 회사·정규화 보고서명·근접기간 기준의 후속 정정·철회 문서와 30건 모두 일치
- 일치율 100%
- 다만 이 휴리스틱은 독립 사건을 섞을 수 있으므로 명시적 원접수번호·정정사항표 기준 재감사가 필요함

### 3. `report_nm` 접두어

다음 공식 접두어 8종을 실제 OpenDART JSON에서 모두 확인했다.

복수 접두어가 붙는 사례도 확인했다. 예: `[정정제출요구][기재정정]`. `[연장결정]`은 E002가 아니라 B 계열 주요사항보고서에서 관찰되어 탐색 유형을 수정했다.

## 계획서와 다른 점 및 결정

`DECISIONS.md`에 다음을 기록했다.

1. 개인용 로컬 환경에서는 정책 근거가 미확인이어도 기술 상태와 검색 품질 향상이 확인되면 DART 본문검색을 사용한다.

4. `rm`은 후보 신호로만 사용하고 명시적 근거로 최종 검증한다.
5. `[연장결정]` 표본 탐색은 E002가 아니라 B 공시유형으로 라우팅한다.
6. D004 공시 건수와 공개매수 사건 수를 분리한다.
7. `013` 실측 결론은 목록 API의 정상 무데이터에 한정한다.

### 실제 인수검증에서 발견·수정한 문제

- 프로젝트 비공개 경로 `_local_data/.env`에 사용자가 제공한 OpenDART 키를 저장했다. `_local_data/`는 Git에서 제외되며 키 값은 응답·로그·커밋에 포함하지 않는다.
- Python 3.14의 `VERIFY_X509_STRICT`가 DART의 legacy CA BasicConstraints와 충돌해 실제 네트워크 호출이 실패했다. 단계 0 프로브와 동일하게 strict 플래그만 해제하고 `CERT_REQUIRED`와 호스트명 검증을 유지하도록 공통 HTTP 클라이언트를 수정했다.
- 삼성전자 요청에 다른 회사 결과가 섞이는 문제를 발견했다. 회사명을 CORPCODE로 먼저 확정하고 DART `textCrpCik`·`b_textCrpCik`과 OpenDART `corp_code`에 같은 8자리 코드를 사용하도록 수정했다.
- 주요사항보고서 본문 문구가 포함된 결과보고서가 오탐으로 들어오는 문제를 발견했다. 명시적 `주요사항보고서` 요청은 구조화된 보고서명 접두어로 제한하고 OpenDART 공시유형 `B` 목록으로 전기간을 보강하도록 수정했다.
